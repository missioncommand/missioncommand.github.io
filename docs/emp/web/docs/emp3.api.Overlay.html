Class: Overlay

emp3.api. Overlay

This class handles all operations on a Geo Container object or Overlay.

new Overlay( [args])

This default constructor creates an Overlay and its encapsulated GeoContainer.

Parameters:
Name Type Argument Description
args <optional>

Arguments object

Properties
Name Type Argument Description
name string <optional>
description string <optional>
geoId string <optional>

This default constructor creates an Overlay and its encapsulated GeoContainer and assigns it
the id provided. If args.geoId is specified to not specify args.container.
Examples

A basic overlay using defaults

var overlay = new emp3.api.Overlay();

With a name and definite geoId

var unitsOverlay = new emp3.api.Overlay({
  name: "Hostile Ground Units",
  geoId: "hgu_1_12271500Z2016",
  description: "Hostile ground units positions on 12/27/2016 as of 1500Z"
});

dataProviderId :string

This is a free form string that can represent an ID specific to this piece of data as it is tracked uniquely by the system providing this data. This is different from GeoId as the GeoId is of type UUID. Some systems may use IDs formatted in a way that it cannot be stored in a UUID. In this case, this property can be used by the system to track this psece of data by the ID they use internally. The GeoID proerty will be used by CMAPI as the unique ID and this ID is for the benefit of the system providing the data.

Type:
  • string
Inherited From:

description :string

A simple string, or HTML formatted string that can be displayed to describe the IGeoBase. In the case of a feature the description property can be used to store the content that should display in a pop up window after clicking on a feature.

Type:

Inherited From:

geoId :string

The id value should be a UUID @see https://www.ietf.org/rfc/rfc4122.txt to avoid conflict of data from multiple sources. All id values SHALL be unique regardless of type and conflicting id values will be treated as the same element. Once this ID is set, it should not be changed to avoid refernces by ID to break. The setter is provided so that an item can be deserialized from an object with an exisitng ID or provided an exisiting ID immediately after instantiation.

types.

Examples

Adding a container event to a map

var added = map.addEventListener({
   eventType: emp3.api.enums.EventType.CONTAINER
   callback: myListener
});

function myListener(event e) {
    // e is an OverlayAddedEvent in this case.
    console.log(e.overlay.name);
}

Adding an overlay change event listener to an overlay

overlay.addEventListener({
  event: emp3.api.enums.EventType.CONTAINER
  callback: function(event) {
    // do something with the event
  }
 });

Adding a feature interaction event listener to a feature

feature.addEventListener({
  event: emp3.api.EventType.FEATURE_INTERACTION
  callback: function(event) {
    switch (event.type) {
      case emp3.api.enums.UserInteractionEventEnum.LONG_PRESS:
        // handle long press
        break;
      default:
        // do nothing
    }
  }
});

apply( [args])

This method triggers an update of the overlay.
The only properties which may be updated in the fashion are the name and description.
Changing the properties locally without calling apply will cause updates to be lost.

Parameters:
Name Type Argument Description
args object <optional>
Properties
Name Type Argument Description
onSuccess emp3.api.Overlay~applyCallback <optional>
onError emp3.api.Overlay~errorCallback <optional>
Deprecated:
  • Use Overlay.update
Example

Renaming an overlay

var weatherOverlay = new emp3.api.Overlay({name: 'wetter map', geoId: 'weatherMap'});
map.addOverlay({
  overlay: weatherOverlay,
  onSuccess: function() {
    // fix spelling of name
    weatherOverlay.name = "weather map";
    weatherOverlay.apply();
  }
});

removeEventListener(args)

Removes an event listener from the container. Event listeners must be removed
before destroying map instances to prevent a memory leak.

Parameters:
Name Type Description
args Object

Parameters are provided as members of the args object.

Properties
Name Type Description
eventType emp3.api.enums.EventType

The event to remove.
callback function

Function to remove, this will be returned from the initial call to addEventListener
Inherited From:
  • emp3.api.Container#removeEventListener
Example

Adding a container event listener to a map

var map = new emp3.api.Map(...);

var added = map.addEventListener({
  event: emp3.api.enums.EventType.CONTAINER,
  callback: function() {
    // do something;
  }
});

map.removeEventListener({
  event: emp3.api.enums.EventType.CONTAINER,
  callback: added
});
